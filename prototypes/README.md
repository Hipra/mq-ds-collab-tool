# Prototypes Directory

This directory contains MUI-based React prototypes generated by Claude Code.

## Prototype Convention

Each prototype is a folder named by its ID (used in URLs):

```
prototypes/
├── my-design/
│   ├── index.jsx       # Required — the React component
│   └── metadata.json   # Optional — name, createdAt, etc.
└── sample/
    ├── index.jsx
    └── metadata.json
```

## URL Pattern

- Preview: `/preview/<id>` (e.g., `/preview/my-design`)
- Bundle API: `/api/preview/<id>/bundle`

## index.jsx Requirements

Your prototype file MUST have a default-exported React component:

```jsx
import { Box, Typography, Button } from '@mui/material';

export default function MyDesign() {
  return (
    <Box sx={{ p: 3 }}>
      <Typography variant="h4">My Design</Typography>
      <Button variant="contained">Click me</Button>
    </Box>
  );
}
```

**Rules:**
1. Use `export default function ComponentName()` — the default export is required
2. Import MUI components from `@mui/material` only (they are provided via CDN import map)
3. Import React hooks from `react` (e.g., `import { useState } from 'react'`)
4. Do NOT import from relative paths or local node_modules — this prototype runs in the browser

## Claude Code Instructions

When generating a prototype, use this system prompt addition:

> Generate a single React component in a file called `index.jsx` inside the `prototypes/<id>/` directory.
> The component MUST use `export default function ComponentName()` syntax.
> Import only from `@mui/material`, `@emotion/react`, and `react`.
> Do not add local state unless demonstrating interactive behavior.
